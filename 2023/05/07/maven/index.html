maven项目结构

于python中pip不同的是，maven功能更多，如

  • 提供了一套标准化的项目结构；
  • 提供了一套标准化的构建流程（编译，测试，打包，发布……）；
  • 提供了一套依赖管理机制。

maven项目都有固定的格式，如果每一个项目都自己搞一套配置，肯定会一团糟。

pom.xml

其中最关键的是

说明了这个项目依赖了什么。其中groupId类似于Java的包名，通常是公司或组织名称，artifactId类似于Java的类名，通常是项目名称，再加上version。

maven下载依赖不是如pip一般使用命令，而是在这个pom.xml中填写好相关依赖，然后maven就会自动下载这个依赖包。

这个就被称为maven的lifecycle

maven不仅是一个依赖下载工具，还可以对项目进行管理。

使用的这些命令被称为phase，注意命令也是执行上文的流程，只是根据具体的命令，决定执行到那一步。

如果我们运行mvn package，Maven就会执行default生命周期，它会从开始一直运行到package这个phase为止：

  • validate
  • …
  • package
